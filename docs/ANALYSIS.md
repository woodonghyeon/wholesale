## 1. API 활용 추천 기능

### 🟢 즉시 구현 가능 (보유 API 기준)

| 기능 | 활용 API | 설명 |
|------|----------|------|
| **상품 자동 등록** | 네이버 쇼핑 | 바코드/상품명으로 정보+이미지 자동 채움 (구현 완료) |

| **고객 구매 패턴 분석** | Supabase | 고객별 재구매 주기 계산 → 이탈 위험 고객 알림 |

## 2. 시스템 로직 부족 분석

### 🔴 높은 우선순위

#### 2-1. 재고 음수 허용 문제
- **현상**: 판매 시 재고 검증 없이 차감 → 재고가 음수가 될 수 있음
- **해결**: `adjust_inventory` RPC에서 `quantity + delta < 0` 시 에러 반환 또는 경고
- **영향**: 실제 재고와 시스템 재고 불일치 → 실사 시 혼란

#### 2-2. 거래명세서-재고 연동 미완성
- **현상**: 거래명세서 작성 시 재고 차감이 수동/자동 혼재
- **해결**: 매출 전표 저장 시 `slip_items`의 각 품목을 자동으로 재고 차감
- **영향**: 재고 현황 신뢰도 저하

#### 2-3. 반품 처리 흐름 부재
- **현상**: 반품 관리 페이지가 있지만 반품 시 재고 환입·매출 취소·미수금 처리가 연동 안 됨
- **해결**: 반품 확정 시 → 재고 복구 + 역전표 자동 생성

#### 2-4. 미수금 자동 생성 미구현
- **현상**: 거래명세서 작성 후 미수금이 자동으로 생성되지 않음
- **해결**: 전표 저장 시 `receivables` 테이블에 자동 레코드 생성

### 🟡 중간 우선순위

#### 2-5. 부가세 계산 로직 불일치
- **현상**: 공급가액 = `unit_price × qty / 1.1` (역산) → 절사 오류 누적
- **해결**: 상품 등록 시 과세/면세 구분 필드 추가, 정확한 세액 계산

#### 2-6. 사업자별 데이터 격리 미완성
- **현상**: 일부 API 엔드포인트가 `business_id` 필터 없이 전체 데이터 반환
- **해결**: 모든 쿼리에 `business_id` 필터 일관 적용 + RLS 정책 점검

#### 2-7. 동시성 처리 없음
- **현상**: 바코드 스캔으로 동시에 재고 차감 시 race condition 가능
- **해결**: Supabase RPC에서 행 단위 락 사용

#### 2-8. 상품 번들(묶음) 로직 미구현
- **현상**: `is_bundle` 필드는 있지만 번들 구성 품목 테이블이 없음
- **해결**: `bundle_items` 테이블 추가, 번들 판매 시 구성 품목 재고 자동 차감

### 🟢 낮은 우선순위

#### 2-9. 페이지네이션 서버사이드 미적용
- **현상**: 상품 358개를 전부 클라이언트로 받아서 JS로 필터링
- **해결**: Supabase `.range()` 사용한 서버사이드 페이지네이션

#### 2-10. 이미지 외부 URL 의존성
- **현상**: 네이버 이미지 URL은 만료되거나 변경될 수 있음
- **해결**: Supabase Storage에 이미지 업로드 후 영구 URL 사용

## 3. 자사몰 연동 설계

### 권장 아키텍처

```
자사몰 (Next.js / Shopify 등)
       ↕ REST API / 웹훅
이 시스템 (도매 관리)
       ↕
  Supabase DB (단일 소스)
```

### 구현 순서

1. **API 레이어 구축** — `/api/external/` 하위에 자사몰용 엔드포인트
   - `GET /api/external/products` — 재고 포함 상품 목록
   - `GET /api/external/inventory` — 실시간 재고 조회
   - `POST /api/external/orders` — 자사몰 주문 수신 웹훅

2. **API Key 인증** — `x-api-key` 헤더 기반 서버 간 인증

3. **채널 추가** — 기존 `channels` 테이블에 '자사몰' 채널 추가 → 채널별 매출 자동 집계

4. **재고 통합** — 네이버/자사몰/오프라인 주문 모두 동일 `inventory` 테이블 차감
